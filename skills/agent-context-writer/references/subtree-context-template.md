# Subtree Context Template

子目录上下文只负责组织局部内容，不负责决定工具名、文件名或路径。它的职责是补上“进入这个子树后才需要知道”的局部知识。

优先写边界、局部结构、局部命令、局部验证和局部禁区。只保留对该目录内工作有直接帮助的信息。

通常会覆盖这些内容：

1. 作用域说明
2. 子树结构
3. 局部工作流
4. 局部约定
5. 局部风险和禁区

可以按下面的骨架组织：

```md
# <目录或模块名>

## 作用域
- 这个文件适用于哪些路径
- 这个目录在整个项目中的职责

## 结构
- `index.ts` / `main.py`: ...
- `components/`: ...
- `services/`: ...
- `tests/`: ...

## 局部工作流
- 在这里改代码通常先看哪些文件
- 局部开发或测试命令
- 只有这个目录会用到的脚本、fixture、mock、生成流程

## 局部约定
- 这个模块内部的重要抽象和边界
- 哪些模式应延续，哪些写法应避免
- 相比根目录规则，这里有哪些额外限制或例外

## 风险提示
- 哪些文件改动影响面大
- 哪些兼容性、数据结构、协议、接口不能随意改
```

写作时注意：

- 第一段就说明该文件适用于哪个目录范围
- 如果这里依赖根目录上下文文件，不要复制根目录命令和通用约定
- 如果局部目录没有独立命令，就不要硬写“局部命令”一节
- 如果某个限制只对这个子树生效，要明确写出，不要混成全局规则
- 如果用户是根据某个具体文件触发任务，模板内容应围绕该文件所在模块，而不是只围绕单个文件本身
